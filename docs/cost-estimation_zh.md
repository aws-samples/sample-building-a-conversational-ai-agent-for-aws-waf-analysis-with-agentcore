# 成本估算

[English](cost-estimation.md)

WAF Agent 部署的月度成本估算。所有价格为美元。

## 成本组成

### 1. 基础设施（固定月费）

| 组件 | 成本 | 说明 |
|---|---|---|
| CloudFront（前端） | **$0** | 免费套餐：1 TB 出站 + 1000万请求/月。内部小工具不会超出。 |
| AWS WAF（前端防护） | **~$8/月** | 1 个 WebACL ($5) + 3 条规则 ($3)。请求费用对内部使用可忽略。 |
| Cognito 用户池 | **$0** | 免费套餐：10,000 MAU。内部团队不会超出。 |
| S3（前端托管） | **<$0.10/月** | ~1 MB 静态文件。 |
| **小计** | **~$8/月** | |

### 2. AgentCore 计算（按会话）

AgentCore 仅对活跃 CPU 时间计费。I/O 等待（等待 LLM、API 调用）免费。

| 指标 | 典型值 |
|---|---|
| 会话时长 | 30–120 秒 |
| 活跃 CPU 时间 | 约为会话时长的 30%（其余为 I/O 等待） |
| 峰值内存 | ~500 MB |

**每次会话成本：**
- CPU：30秒活跃 × 1 vCPU × ($0.0895/3600) = $0.00075
- 内存：120秒 × 0.5 GB × ($0.00945/3600) = $0.00016
- **合计：~$0.001/次会话**

| 月使用量 | 会话数 | AgentCore 成本 |
|---|---|---|
| 轻度（1名工程师，5次/天） | ~150 | **$0.15** |
| 中度（5名工程师，10次/天） | ~1,500 | **$1.50** |
| 重度（20名工程师，20次/天） | ~12,000 | **$12.00** |

### 3. LLM Token 成本（最大组成部分）

Claude Sonnet 4 在 Amazon Bedrock 上的大致定价：
- 输入：~$3.00/百万 token
- 输出：~$15.00/百万 token

**典型会话 token 用量：**

| 场景 | 输入 token | 输出 token | 成本 |
|---|---|---|---|
| 简单查询（列出 WebACL、指标） | ~5K | ~2K | **$0.045** |
| 调查（绕过检测，10次工具调用） | ~30K | ~5K | **$0.165** |
| 深度分析（13次工具调用 + 报告） | ~50K | ~8K | **$0.270** |
| ROI 报告生成 | ~40K | ~10K | **$0.270** |

**月度估算：**

| 使用级别 | 会话数 × 平均成本 | 月 token 成本 |
|---|---|---|
| 轻度（150次会话，多为简单查询） | 150 × $0.10 | **$15** |
| 中度（1,500次会话，混合） | 1,500 × $0.15 | **$225** |
| 重度（12,000次会话，混合） | 12,000 × $0.15 | **$1,800** |

### 4. CloudWatch 成本

**指标（GetMetricData）：** $0.01/1,000个指标请求
- 典型会话：5–20次指标查询 → $0.0001–$0.0002/次会话
- **月度：<$1**，即使重度使用

**Logs Insights：** $0.005/GB 扫描
- 典型查询扫描 50–500 MB 日志
- 调查会话（10次查询）：~2 GB 扫描 → $0.01
- **月度（中度使用）：** 1,500次会话 × $0.01 = **$15**

### 5. Athena（仅 S3 日志）

$5.00/TB 扫描。仅在 AWS WAF 日志发送到 S3（而非 CloudWatch Logs）时使用。

- 典型查询：10–100 MB 扫描 → $0.0001–$0.0005
- DDL 查询（CREATE/DROP TABLE）：**免费**
- **月度（如使用）：** <$5

### 6. AgentCore Memory（可选）

仅在启用 Memory 时产生费用（设置了 `MEMORY_ID` 参数）。

| 维度 | 价格 |
|---|---|
| 短期记忆（CreateEvent） | $0.25/千次请求 |
| 长期记忆存储 | $0.75/千条记录/月 |
| 长期记忆检索 | $0.50/千次请求 |

**典型每次会话用量：** 5–15 次 CreateEvent（每轮对话一次）+ 1 次 RetrieveMemoryRecords（会话开始时）。

| 使用级别 | 月事件数 | 月成本 |
|---|---|---|
| 轻度（150次会话） | ~1,500 事件 + 150 检索 | **<$1** |
| 中度（1,500次会话） | ~15,000 事件 + 1,500 检索 | **~$5** |
| 重度（12,000次会话） | ~120,000 事件 + 12,000 检索 | **~$36** |

### 7. DynamoDB（会话历史）

按需计费。存储完整对话消息，30 天 TTL 自动过期。

| 维度 | 价格 |
|---|---|
| 写请求单位 (WRU) | $1.25/百万次 |
| 读请求单位 (RRU) | $0.25/百万次 |
| 存储 | $0.25/GB/月 |

**典型每次会话用量：** ~10 次写入（消息）+ 1 次读取（恢复历史）+ 元数据更新。

| 使用级别 | 月写入 | 月读取 | 存储 | 月成本 |
|---|---|---|---|---|
| 轻度（150次会话） | ~1,500 | ~200 | <1 MB | **<$0.01** |
| 中度（1,500次会话） | ~15,000 | ~2,000 | ~10 MB | **<$0.01** |
| 重度（12,000次会话） | ~120,000 | ~15,000 | ~100 MB | **~$0.05** |

DynamoDB 在此用例下任何规模的成本都可忽略。

### 8. 会话 API — Lambda + API Gateway（可选）

仅在部署了会话历史功能（`deploy/sessions-api.yaml`）时产生费用。

| 组件 | 价格 |
|---|---|
| Lambda | 免费套餐：每月 100 万次请求 + 40 万 GB 秒。内部工具用量远低于免费额度。 |
| API Gateway HTTP API | $1.00/百万次请求 |

**典型用量：** 每次会话 ~10 次 API 调用（列表 + 恢复）。所有使用级别都在 Lambda 免费套餐内。API Gateway 成本 < $0.01/月。

## 月度总成本估算

| 使用级别 | 基础设施 | AgentCore | Token | CloudWatch | Memory | DynamoDB | 合计 |
|---|---|---|---|---|---|---|---|
| **轻度**（1名工程师） | $8 | $0.15 | $15 | $1 | $1 | <$0.01 | **~$25/月** |
| **中度**（5名工程师） | $8 | $1.50 | $225 | $15 | $5 | <$0.01 | **~$255/月** |
| **重度**（20名工程师） | $8 | $12 | $1,800 | $50 | $36 | $0.05 | **~$1,900/月** |

## 关键结论

1. **Token 成本占主导。** 基础设施和计算费用可忽略。主要成本驱动因素是 LLM 使用量。
2. **AgentCore 极其便宜。** ~$0.001/次会话，因为只对活跃 CPU 计费（I/O 等待免费）。
3. **CloudWatch Logs Insights 很便宜。** $0.005/GB 扫描。即使重度使用也不超过 $50/月。
4. **免费套餐覆盖基础设施。** CloudFront、Cognito 和基本 S3 对内部工具实际上免费。
5. **DynamoDB 实际上免费。** 按需计费下 <100 万次请求/月不到 $1。
6. **成本随使用量线性增长。** 无最低承诺或预留容量。

## 成本优化建议

- 优先使用**指标而非日志**——指标查询比日志查询便宜 100 倍
- 保持调查**时间窗口窄**（≤6小时）——减少扫描的 GB 数
- 对历史分析使用 **Athena**（S3 日志）——比在 CloudWatch 中保留数月日志更便宜
- 考虑对定时报告使用 **Amazon Bedrock 批量推理**（token 成本打5折）
- Agent 的 system prompt 使用约 1,200 token——这是每次会话的固定成本，无法进一步降低
